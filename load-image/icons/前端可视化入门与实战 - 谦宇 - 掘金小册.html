这里可能有小伙伴会问了：clip 和 clip-path 的区别是什么？

我们这里使用 clip，其实 clip 已经被 CSS3 弃用，在使用 clip 裁剪的时候我们必须使用绝对定位，但在 clip-path 中我们不需要绝对定位，就可以裁剪出圆形、椭圆等形状。

css
复制代码
/* 裁剪矩形 */
clip-path: inset(10px 20px 30px 40px);
/* 裁剪多边形, polygon的每组代表每个点的坐标百分比位置 */
clip-path: polygon(50% 0%, 90% 20%, 100% 60%, 75% 100%, 25% 100%, 0% 60%, 10% 20%);

position: absolute;
clip: rect(110px, 170px, 160px, 60px);

对于 clip 中的参数，rect() 中代表矩形的坐标参数依次表示上边界、右边界、下边界和左边界，就像下图生动地做了描述：

既然知道了 clip-path 的好处，那么接下来我们做个替换：

css
复制代码
clip: rect(0px, 75px, 150px, 0px);
->
clip-path: polygon(0% 0%, 50% 0%, 50% 100%, 0% 100%);

为什么需要裁剪两次呢？

因为.pie 类的剪切，将元素的左半部分剪切掉，只显示右半部分。这样一来，.pie 元素就变成了一个半圆形。.hold 类的剪切将元素的右半部分剪切掉，只显示左半部分。这样一来，.hold 元素就变成了一个与 .pie 元素相对应的半圆形。这么做就是因为矩形裁剪是单方向的，一旦圆环超过半圆就无法裁剪了。所以需要裁剪两次。

2. conic-gradient 圆锥渐变

第一种方案由于基于裁剪很显然分割是很明确的，导致我们无法实现渐变，尤其当需求需要渐变表示递进的强弱关系时，我们就可以使用圆锥渐变轻松实现。

实现思路：我们通过计算环的周长，总长其实就是 stroke-dasharray，通过 strokeDashoffset 来偏移我们的虚线线段，那么开始的就是我们的实线线段。其实就是一个蚂蚁线。让这个线长度等于我们的环长度，通过 api 让实线在开始的位置。下图白色环状区域就是我们的 strokeDashoffset 的偏移量。最后圆锥渐变可以轻松定义我们的百分比的渐变程度，比如 0 到 70% 是某个颜色，70% 到 100% 是某个颜色，等等。

圆锥渐变最大的问题就是会有兼容问题，比如在 Firefox 和 Internet Explorer 中会不支持。除此之外，在 Safari 浏览器上会有一个很大的问题，就是 Safari 的 dasharray 的实线和虚线的第一个元素是不是实线，而在 Chrome 中第一个是虚线，这里就需要做个简单的兼容了。

js
复制代码
/**
 *设置百分比
 * @param percent 偏移白色环状的量
 */
function setProgress(percent) {
    const progress = (percent / 100) * innerCircumference
    const userAgent = navigator.userAgent
    if (userAgent.indexOf('Safari') > -1 && userAgent.indexOf('Chrome') < 0) {
        circle.style.strokeDashoffset = (outerCircumference + (outerCircumference - progress)) + ''
    } else {
        circle.style.strokeDashoffset = -(progress) + ''
    }
}

这里需要注意下这段 CSS transform: perspective(12000px) translateY(400px) rotateY(-10deg);，因为我们的默认视点是屏幕中心，这就意味着我们想看到立方体的顶面，就必须设置translateY()，为了展示右侧面我们必须设置rotateY()，这个时候一定是一个透视视图而不是正交视图，所以我们将设置 perspective 为一个非常大的值，来矫正成一个正交的视图。如果不设置 perspective，没有透视效果，意味着你的柱子是会看不到顶面的。

总结

我们总结下各个图表用 CSS 实现的思路：

  • 柱状图：通过 flex 布局 + label（attr 属性 + 伪元素 + html 自定义属性）。
  • 饼图和圆环图： 一种是 Clip 裁剪 + transform 变换，另一种是通过conic-gradient 渐变来实现。
  • 散点图：图表定位元素，元素根据 bottom、left 对应 x、y 坐标做对应偏移。
  • 折线图：核心在于var angle = Math.acos(对边 / 斜边) * (180 / Math.PI); 的计算角度值并 rotate 即可。
  • 3D 柱状图：通过 tranfrom 的 perspective 绘制。
